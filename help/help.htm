Quoteoftheday_XH 1.1.1

For CMSimple_XH >= 1.5.4 or CMSimple 4.x

"Quote of the day" creates a div box in which a quote is placed. This quote comes from one of several quote collections entered by the user in the backend. The quote is exchanged daily, weekly or monthly, at random or in a fixed order.

The quotes are entered as plain text with simplified markup similar to wiki software.

New in version 1.1

  • CSS-frames are now fully responsive. Quotes look good also in smartphones
  • Default saving location for quotes now in the userfiles. Saving location can be set in config with option list to userfiles/ plugin (default up to Quoteoftheday 1.0)/ or as subfolder of the content folder.
  • Small improvements in admin view.

New in version 1.0

  • Quotes can contain links, images and audio files.
    Links from a secondary language differ from links to the main language. These links are created at the beginning of the day when the plugin selects the quote. If you use the same quote in main and secondary language, the link will only work in one language, i.e. the language in which the quote is generated first. You can however copy the quote file to a new name and use two identical files, each for one language only.
  • Additional formatting possibilities
  • Selectable frames for the quotes
  • Filebrowser integration: The Quoteoftheday zip file contains also a file to be inserted in the filebrowser folder at plugins/filebrowser/editorhooks/quoteoftheday/ to manage the filename transfer. The unzipped Quoteoftheday file is a ready to use "plugins" folder to be inserted into the website base.
  • Single quote view with preview, reordering and deleting possibility, next to the usual total view of all quotes of a quote file.
  • Bugfix (0.8 + 0.9 would revert to the standard quote file after preview. Now fixed.)

Update to 1.1

Saving your old installed plugin before updating may always be prudent.

Install the unzipped download into the base folder of CMSimple_XH. This will overwrite your old quoteofthedays plugin, add a file to the filebrowser plugin and add a folder for saving quotes to the userfiles. Your existing quote files should remain intact. Config settings have to be entered again.

In case you made some changes to the css, save you old css file before the update and create later your custom css class as detailed in the next section.

Updates from 0.5 and earlier version need entering after every quote the new quote separator === and the trigger for the author line -.

Quote Frames

Quotes are displayed in a frame, which is a div container with a selectable class. The class name is saved in the quote file header. You can easily create additional frame classes: Just enter a new class in the stylesheet.css, the name has to start with .quote_.

Sometimes it is easier to edit one of the existing frame classes, e.g. by changing the value of max-width or adding a font color with color:black;. To prevent floating images overlapping the bottom quote border, add clear:both;. However in some templates this will affect the design, so a decision has to be made per case.

Calling Quoteoftheday

  • on a page in XH 1.6.5 or higher:
    {{{quoteoftheday}}} oder {{{quoteoftheday 'Dateiname'}}}. The filename ending .txt doesn't have to be entered.
    In older XH versions the old style method has to be used: {{{PLUGIN:quoteoftheday();}}} or {{{PLUGIN:quoteoftheday('FILENAME');}}}.
    Important: In XH 1.6.5 and 1.6.6 there was a bug (fixed in 1.6.7), which invalidated a new style plugin call without "PLUGIN:" when on the same page another old style call with "PLUGIN:" was present. So either update to 1.6.7 or don't mix old and new calling styles on the same page.
  • in the template:
    <?php echo quoteoftheday();?> or <?php echo quoteoftheday('FILENAME');?>

Entering and editing quotes

Preset is the language specific quote file, e.g. for English quote_en.txt. Instead of selecting "Standard quote file" you can select quote_en.txt, which amounts to the same. You can create or delete quote files via the options menu. File names may not contain accented letters or spaces.
